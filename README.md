# 自動車保険見積もりシステム (Car Insurance Quote System)

## 1. プロジェクト概要 (Overview)
本プロジェクトは、Spring Boot + React + PostgreSQL をベースとした自動車保険のオンライン見積もりシステムです。（課題用）
ユーザーが自身の車両情報や運転者情報を入力することで、リアルタイムに保険料の見積もり結果を算出・表示します。また、管理者向けのログイン機能や過去の見積もり履歴を閲覧できるダッシュボード機能も提供します。

## 2. 技術スタック (Tech Stack)

### Backend
- Java 21
- Spring Boot 3
- Spring Data JPA, Spring Security, Spring Web
- Hibernate Validator (入力検証)
- Maven
- Swagger / OpenAPI (APIドキュメント自動生成)

### Frontend

- TypeScript
- Tailwind CSS (UIスタイリング)
- React Router (ルーティング)
- Axios (API通信)

### Database
- PostgreSQL 16

### Infrastructure / DevOps
- Docker & Docker Compose (コンテナ化)

## 3. 起動手順 (Getting Started)
本プロジェクトは Docker Compose を用いて、フロントエンド、バックエンド、データベースを一括で起動することができます。

### 前提条件
- Docker および Docker Compose がインストールされていること

### 起動コマンド
プロジェクトのルートディレクトリで以下のコマンドを実行します：
```bash
docker compose up --build -d
```

### アクセス先URL
- **フロントエンド (UI)**
  - 一般ユーザー用: [http://localhost:5173](http://localhost:5173)
  - 管理者ログイン: [http://localhost:5173/admin/login](http://localhost:5173/admin/login)
- **バックエンド (API)**: [http://localhost:8080/api](http://localhost:8080/api)
- **Swagger API ドキュメント**: [http://localhost:8080/swagger-ui/index.html](http://localhost:8080/swagger-ui/index.html)

### 初期管理者アカウント
管理者用画面にアクセスする際の初期アカウントは以下の通りです：
- **ユーザー名**: `admin`
- **パスワード**: `password`

## 5. アーキテクチャ・設計上の工夫点 (Design & Architecture)

本プロジェクトの開発において、以下の優れたプラクティスを取り入れています：
- **フロントエンドとバックエンドの徹底した分離**: React を用いた SPA (Single Page Application) と Spring Boot の REST API により、責任境界を明確にし、拡張性を確保しています。
- **Docker コンテナ化**: 環境構築の手間を省き、すべての開発者がコマンドひとつで同一の環境を構築し、開発・実行できるよう `docker-compose.yml` を整備しました。
- **堅牢なバックエンドバリデーション**: フロントエンドでの単なる形式的な入力チェックに加え、バックエンドでは JPA および Hibernate Validator を活用し、DBの制約と合わせた厳格なデータ検証を行っています。さらに、カスタムバリデーション（@AssertTrue等）を用いて「未来の初度登録年月を弾く」といった業務ロジックレベルの深い検証まで実装し、データの完全性を担保しています。
- **UIテストカバー率の向上**: フロントエンドでは React Testing Library (RTL) を用い、ユーザー視点での操作や画面表示のコンポーネントテストを実装しています。
- **API ドキュメントの自動生成**: SpringDoc (Swagger) を導入し、バックエンドの実装と同期した最新の API 仕様を動的に確認できる環境を構築しました。

## 6. 制約事項 (Limitations)
- 本システムは課題デモンストレーション用であり、実際の保険会社の決済システムや外部システム（ナンバープレートからの車両情報照会など）の API とは連携していません。
- データの永続化や認証の仕組みは実装されていますが、本番環境レベルの厳密なセキュリティ監査やチューニング等は実施していません。

## 7. 今後の改善提案・パフォーマンス・セキュリティ

システムのさらなる発展と実用化に向けて、以下の改善事項を推奨します。

### システムの改善提案 (System Improvements)
- **外部APIとの連携による車両情報の自動入力**: ナンバープレート情報や車台番号を入力するだけで、車種や年式などの車両情報を外部の車両データベースAPIから自動取得・補完する機能を導入し、ユーザーの入力負荷を軽減します。
- **フォーム入力におけるUI/UXの最適化**: 段階的なステップフォーム（ウィザード形式）の強化、動的な入力支援（サジェストやエラーのリアルタイムフィードバック）、およびモバイル端末に最適化した直感的な入力 UI を提供し、途中離脱率を低下させます。

### パフォーマンス向上 (Performance Optimization)
- **インメモリキャッシュの導入 (Caffeine Cache)**: 見積もり計算時に頻繁に参照され、かつ更新頻度の極めて低い静的マスタデータ（料率マスタなど）に対して、アプリケーション内のローカルキャッシュ（Caffeine等）を導入します。これにより、DBへの問い合わせI/Oを極限まで削減し、高負荷時でも極めて高速なレスポンスを実現します。
- **フロントエンド配信の最適化 (Gzip圧縮)**: Nginxの設定においてGzip圧縮を有効化し、Reactのビルド成果物（HTML/CSS/JS）のファイルサイズを削減することで、初期ロード時間の短縮とフロントエンドのパフォーマンス向上を図ります。
- **Redisによるキャッシュサーバーの導入**: 見積もり計算時に頻繁に参照される保険料率や車種区分データなどの静的・準静的データをメモリ内データベース（Redis）にキャッシュすることで、データベースへの問い合わせ回数を削減し、応答速度を向上させます。
- **データベースインデックスの最適化**: 過去の見積もり履歴検索や管理者ダッシュボードで頻繁に使用されるクエリの検索条件（`user_id` や `created_at` など）に対して適切な複合インデックスを追加し、データ増加に伴うクエリ速度の低下を防止します。

### セキュリティ対策 (Security Measures)
- **JWT（JSON Web Token）認証へのアップグレード**: 現在のベーシック認証から、ステートレスでスケーラブルな JWT 認証にアップグレードすることで、セッション管理の安全性を高めるとともに、将来的なマイクロサービス化や外部サービス連携にも柔軟に対応します。
- **CSRF（クロスサイトリクエストフォージェリ）対策の導入**: Spring Security の CSRF 保護機能を有効化し、フロントエンド（Axios 等）との通信において CSRF トークンを正しく検証する仕組みを導入することで、不正なリクエスト送信によるセキュリティリスクを排除します。
- **HTTPセキュリティヘッダーの強化**: フロントエンドを配信するNginxに X-Frame-Options や Content-Security-Policy などの本番レベルのセキュリティヘッダーを追加し、XSS（クロスサイトスクリプティング）やクリックジャッキングなどの攻撃を未然に防ぎます。
- **セキュアなコンテナ环境の構築 (Distrolessイメージの採用)**: 本番環境のDockerイメージのベースとして、シェルやパッケージマネージャを含まない「Distroless」イメージを採用します。これにより、万が一コンテナ内に侵入された場合でも、悪意のあるスクリプトを実行されるリスクをインフラ層から根本的に排除します。
