## 2. システム概要
本システムは、利用者が自動車保険の見積条件を入力し、課題用の簡易料率ロジックにより年間保険料・月額保険料を算出するWebアプリケーションである。参考サイトの画面構成を参考にするが、文言・ロゴ・デザイン・実料率はコピーしない。 （画面はユーザビリティを最優先しビジュアルデザインの構築は最低限でよい。）

| 分類       | 内容                                                         |
| ---------- | ------------------------------------------------------------ |
| 利用者機能 | 見積条件入力、入力確認、見積結果表示、見積番号発行 。        |
| 管理者機能 | ログイン、見積一覧検索、見積詳細参照、CSV出力。              |
| 対象外     | 実契約申込、決済、本人確認、事故受付、外部保険会社API連携 。 |

| Backend   | Spring Boot        | REST API、サーバー側入力チェック、見積計算、DB永続化、管理者認証。 |

| ロードサービス | あり            | +1,500円  |

### 5.1 計算手順
1. 基本保険料50,000円に各係数を順番に乗算する。
2. 乗算後、加算型の補償・特約金額を加算する。
3. 年間保険料は10円未満を四捨五入する。
4. 月額保険料は年間保険料÷12を行い、10円未満を四捨五入する。
5. 計算に使用した係数・加算額はquote_breakdownsに保存する。

## 6. エラー仕様
| HTTP | コード           | 内容                                             |
| ---- | ---------------- | ------------------------------------------------ |
| 400  | VALIDATION_ERROR | 入力チェックエラー。項目ごとのメッセージを返す。 |
| 401  | UNAUTHORIZED     | 管理者認証が必要。                               |
| 403  | FORBIDDEN        | 権限不足。                                       |
| 404  | NOT_FOUND        | 見積番号が存在しない。                           |
| 500  | SYSTEM_ERROR     | 想定外エラー。詳細なスタックトレースは返さない。 |

## 7. 実装上の制約
- Controller、Service、Repository、Entity、DTOを適切に分離すること。
- パスワードは平文保存しないこと。BCrypt等を使用すること。
- フロント側チェックだけでなく、バックエンド側でも必ず入力チェックすること。
- READMEに起動方法、テスト実行方法、管理者初期IDを記載すること。
- docker compose upで起動できない提出物は原則減点対象とする。
